# Привязка привычек к целям и месячных целей к годовым

## Суть

Каскадная система прогресса:
- **Привычки** (ежедневные отметки) → считают прогресс **месячных целей**
- **Месячные цели** (подзадачи + привычки) → считают прогресс **годовых целей**

## Данные

### Goal (месячная цель)

Новые поля:
```typescript
target?: number                                    // целевое число (3 книги, 20 тренировок)
linkedHabits?: { habitId: string, target: number }[] // привязанные привычки
```

### YearlyGoal (годовая цель)

Новые поля:
```typescript
target?: number                                  // целевое число (100 книг)
linkedMonthlyGoals?: { goalId: string }[]        // привязанные месячные цели
```

## Формула прогресса

### Месячная цель с target

```
completedCount = выполненные подзадачи (count) + сумма(min(привычка.выполнений, привычка.target))
progress = min(completedCount / target * 100, 100)
```

### Годовая цель с target

```
completedCount = свои выполненные этапы (count)
              + сумма(выполненные подзадачи каждой привязанной месячной цели)
progress = min(completedCount / target * 100, 100)
```

### Цели без target (обратная совместимость)

Работают как раньше: progress = выполненные подзадачи / всего подзадач * 100

## UI

### Карточка месячной цели

- Кнопка «Привязать привычку» внизу подзадач
  - Выпадающий список существующих привычек
  - Поле ввода целевого числа
- Привязанная привычка в списке подзадач:
  - Иконка 🔄 вместо чекбокса
  - Название + мини прогресс-бар + «15/20»
  - Автоматический подсчёт (нельзя отметить вручную)
  - Кнопка ✕ для отвязки
- Поле target в форме создания/редактирования цели

### Карточка годовой цели

- Кнопка «Привязать месячную цель» внизу этапов
  - Выпадающий список месячных целей
- Привязанная месячная цель в списке:
  - Иконка 📅
  - Название + прогресс-бар + «2/3 выполнено»
  - Кнопка ✕ для отвязки
- Поле target в форме создания/редактирования

## Поток данных

1. Привязки хранятся в секции `goals` (linkedHabits на месячных, linkedMonthlyGoals на годовых)
2. Хук целей получает функцию подсчёта выполнений привычки из хука привычек
3. При рендере карточки — берём completions из хука привычек и считаем
4. Всё сохраняется автоматически через существующий debounce 300ms
